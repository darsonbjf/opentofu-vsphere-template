# Security Policy

## Supported Versions

This repository is a public template. Security fixes are applied to the `main`
branch.

## Reporting a Vulnerability

Do not open a public issue for suspected secrets, credential exposure, or a
security vulnerability. Use GitHub's private vulnerability reporting when it is
enabled for the repository, or contact the maintainer through a private channel.

Please include:

- affected files or commands
- impact and reproduction steps
- whether credentials or infrastructure details may have been exposed

Rotate any exposed credential before publishing details or logs.
